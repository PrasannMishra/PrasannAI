# Day 2 — Large Language Models & Provider Landscape

---

# Overview

Today's lesson introduces the foundation of modern AI applications: the Large Language Model (LLM).

Rather than learning provider-specific APIs, we focus on understanding the common concepts shared across all modern LLMs. This knowledge will make it easier to work with Anthropic, OpenAI, Ollama, Gemini, and future providers.

---

# Learning Objectives

By the end of today's lesson, you should be able to:

- Explain what a Large Language Model is.
- Understand the concept of tokens.
- Describe the purpose of a context window.
- Compare local and cloud-hosted models.
- Select the right provider for a given application.

---

# What is a Large Language Model?

A Large Language Model is a neural network trained to predict the next token in a sequence.

Although the underlying mathematics is complex, AI engineers primarily interact with LLMs through APIs, prompts, and structured outputs rather than training models directly.

📖 **Handbook Reference**: [Tokens](../../handbook/llms/tokens.mdx)


---

# Tokens

LLMs do not process complete words or sentences.

Instead, they process **tokens**, which are smaller pieces of text.

Token usage affects:

- Cost
- Context length
- Performance
- Memory usage

- `handbook/llms/tokens.mdx`

---

# Context Window

Every LLM has a maximum amount of information it can consider during a single request.

This limit is called the **context window**.

A larger context window enables:

- Longer conversations
- Larger documents
- More code analysis
- Better retrieval workflows

# Cloud Models vs Local Models

## Cloud Models

Examples:

- Claude
- OpenAI
- Gemini

Advantages:

- Highest quality
- Regular updates
- Large context windows
- Advanced reasoning

Trade-offs:

- API cost
- Internet dependency
- Data leaves your machine

---

## Local Models

Examples:

- Ollama
- Qwen
- Llama
- Mistral

Advantages:

- Free after download
- Offline
- Full privacy
- Fast experimentation

Trade-offs:

- Hardware limitations
- Smaller models
- Lower reasoning capability compared to flagship cloud models

# Provider Landscape

During this roadmap we focus primarily on:

| Provider | Primary Use |
|-----------|-------------|
| Anthropic | Production applications and reasoning |
| Ollama | Local experimentation and learning |
| OpenAI | Cross-provider understanding |

The goal is to learn provider-independent architecture so applications can switch models with minimal changes.

# Best Practices

- Learn concepts before SDKs.
- Design provider-independent applications.
- Separate application logic from provider implementations.
- Use local models for experimentation when possible.

---

# Common Misconceptions

❌ "One provider is the best for every task."

Different providers excel in different areas. Architecture should allow flexibility.

---

❌ "Switching providers requires rewriting the application."

Well-designed abstractions isolate provider-specific code.

# Key Takeaways

- LLMs generate text token by token.
- Tokens influence cost and context.
- Context windows determine how much information a model can process.
- Cloud and local models serve different purposes.
- Provider abstraction is essential for maintainable AI systems.
