# AI Learning Hub - Product Requirement Document (v1)

## Vision

Build a local-first Learning Management System (LMS) where all learning content is stored as MDX files.

The application should require **zero code changes** when adding a new lesson.

Adding a lesson should only involve copying a new folder into the `content/` directory.

The application should automatically:

* Detect the lesson
* Index it
* Display it in the dashboard
* Add it to search
* Update progress
* Link it to projects
* Show it in the roadmap

No backend is required for Version 1.

---

# Primary Goals

1. Local-first
2. Git-friendly
3. Offline capable
4. Zero database
5. Zero backend
6. AI-ready architecture
7. Production-quality UI
8. Easily extensible

# Important Rule

The application must never require code changes when adding a new lesson.

Adding:

content/day-042/lesson.mdx

should automatically make it available everywhere.

The application architecture must be content-driven, not route-driven.


# AI Learning Hub - Development Milestones

The project should be built incrementally in well-defined milestones rather than attempting to implement everything at once.

---

# Milestone 1 (Estimated: 2–3 Days)

## Objective

Build the foundation of the learning portal.

## Features

- React + Vite + TypeScript setup
- Tailwind CSS integration
- MDX rendering support
- Auto-discover lessons from the `content/` directory
- Dashboard
- Sidebar navigation
- Lesson page
- Dark / Light mode
- Search functionality
- Progress tracking (LocalStorage)

## Deliverable

A fully functional documentation portal capable of automatically rendering lessons from MDX files.

---

# Milestone 2

## Objective

Enhance the portal with learning management features.

## Features

- Personal Notes
- Bookmarks
- Resources Library
- Projects section
- Cheat Sheets
- Interview Preparation section

## Deliverable

A complete personal learning management system (LMS) for tracking knowledge and resources.

---

# Milestone 3

## Objective

Improve usability and presentation.

## Features

- Export functionality (Markdown / PDF / Progress)
- Mermaid diagram rendering
- Learning analytics
- UI/UX polish
- Performance optimizations

## Deliverable

A polished, production-quality learning portal.

---

# Milestone 4 (Around Day 40–50 of the AI Engineering Roadmap)

## Objective

Transform the learning portal into an AI-powered learning assistant.

## Features

- Local RAG (Retrieval-Augmented Generation) over MDX lessons
- AI Tutor powered by Ollama
- "Ask My Course" feature
- Semantic Search
- Revision Generator
- Quiz Generator

## Deliverable

An AI-powered learning platform capable of answering questions based on the user's own course material and generating personalized revision content.
