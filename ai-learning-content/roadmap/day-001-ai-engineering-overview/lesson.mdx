# Day 1 — AI Engineering Overview

---

# Overview

Software engineering has evolved significantly with the emergence of Large Language Models (LLMs). Modern software engineers are increasingly expected to integrate AI capabilities into applications rather than treating AI as a completely separate domain.

This roadmap is designed specifically for an experienced frontend engineer who wants to transition into AI Engineering by building production-grade AI applications rather than focusing on machine learning research.

The emphasis throughout this roadmap is on practical engineering skills, including architecture, APIs, streaming, retrieval, context engineering, agents, and reusable AI platforms.

---

# Why AI Engineering?

Traditionally, software engineers built applications by writing business logic directly.

Today, many applications combine:

- Traditional business logic
- LLM capabilities
- Retrieval systems
- AI agents
- Tool calling
- Workflow automation

The role of an AI Engineer is to design and build these intelligent systems while maintaining software engineering best practices.

---

# AI Engineer vs Machine Learning Engineer

| AI Engineer | Machine Learning Engineer |
|-------------|---------------------------|
| Integrates LLMs into products | Trains machine learning models |
| Designs AI applications | Designs training pipelines |
| Works with APIs and prompts | Works with datasets and algorithms |
| Builds user-facing AI features | Optimizes models |
| Focuses on architecture and product integration | Focuses on model performance |

For most product companies, AI Engineering is the faster and more practical path for experienced software developers.

---

# Why Your Existing Experience Matters

With over a decade of frontend experience, you already possess many transferable skills:

- System design
- Component architecture
- API integration
- TypeScript
- Performance optimization
- Debugging
- User experience

The primary additions are understanding LLM behavior, AI-specific architecture, prompt design, context management, and orchestration.

---

# The AI Engineering Stack

```mermaid
flowchart TD

User

↓

React Application

↓

AI SDK

↓

AI Backend

↓

AI Core

↓

Provider

↓

Claude / Ollama / OpenAI
```

Each layer has a distinct responsibility:

- **Frontend**: User experience and interaction.
- **AI SDK**: Client-side abstraction for communication and streaming.
- **AI Backend**: Authentication, orchestration, persistence, and APIs.
- **AI Core**: Prompt construction, provider abstraction, context management, and AI workflows.
- **Provider**: Claude, Ollama, OpenAI, or other LLMs.

# Why Start with Anthropic?

Your organization already uses Claude extensively, making it valuable to learn Anthropic's ecosystem first. The concepts learned throughout this roadmap are provider-independent, allowing you to switch providers with minimal architectural changes.

---

# Local AI with Ollama

To reduce API costs while learning, we chose to use Ollama for local experimentation.

Benefits include:

- No API costs
- Offline development
- Faster iteration
- Safe experimentation
- Local privacy

This enables hands-on practice without consuming paid API credits.

# Common Misconceptions

❌ AI Engineers need to train models from scratch.

Reality: Most AI Engineers integrate existing foundation models into products.

---

❌ Learning one provider locks you into that ecosystem.

Reality: Well-designed abstractions allow switching providers with minimal effort.
